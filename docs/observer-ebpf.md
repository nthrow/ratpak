# `internal/observer/ebpf`

The eBPF-backed implementation of the filesystem observer.

## Files

- `openat.bpf.c` — kernel-side program. Compiled to BPF bytecode by clang at `go generate` time and embedded into the Go binary.
- `observer.go` — userspace-side. Loads the program, attaches to four tracepoints, decodes the ring buffer, emits `observer.Event`s.
- `openat_<arch>_bpfel.{go,o}` — generated by `bpf2go`; `.gitignored`.

## Kernel side (`openat.bpf.c`)

Two maps and four programs.

### Maps

- `events` — `BPF_MAP_TYPE_RINGBUF`, 1 MiB. The kernel→userspace event channel.
- `pending` — `BPF_MAP_TYPE_LRU_HASH`, 8192 entries, key = tid (u32), value = userspace pointer (u64). Per-thread scratch used to pass the filename argument from the enter program to the exit program.

### Programs

- `trace_openat_enter` (tp/syscalls/sys_enter_openat)
- `trace_openat_exit`  (tp/syscalls/sys_exit_openat)
- `trace_openat2_enter` (tp/syscalls/sys_enter_openat2)
- `trace_openat2_exit`  (tp/syscalls/sys_exit_openat2)

Both `_enter` programs stash the filename pointer (arg index 1) keyed by tid in the `pending` LRU. Both `_exit` programs look up that pointer and, if `ctx->ret >= 0`, copy `{pid, tgid, comm, filename}` into a ringbuf record.

The LRU policy means that if a thread exits between enter and exit (e.g. SIGKILL during an openat), its stashed entry will eventually be evicted instead of permanently leaking.

The tracepoint context structs (`sys_enter_args`, `sys_exit_args`) are hand-defined to match the kernel's stable per-syscall format file layout, so we don't need vmlinux.h or BTF.

## Userspace side (`observer.go`)

Generated bindings via:

```go
//go:generate go run github.com/cilium/ebpf/cmd/bpf2go -cc clang -cflags "-O2 -g -Wall" -target amd64,arm64 openat openat.bpf.c
```

`bpf2go` builds a `*_bpfel.o` per target arch (amd64, arm64) and an embedding `.go` file that wraps each one. Generated symbols live alongside `observer.go` in the same package.

`Observe` does:

1. `rlimit.RemoveMemlock()` — soft, only fatal on pre-5.11 kernels.
2. `loadOpenatObjects` — load the embedded BPF object into the kernel.
3. Attach all four syscall tracepoints via `link.Tracepoint`.
4. Open a ringbuf reader on the `events` map.
5. Spawn a goroutine that reads records, decodes each into a `rawEvent` (matches the C struct), and emits an `observer.Event` on the returned channel.

Cancellation: when ctx fires, the reader is closed (which unblocks `rd.Read` with `ringbuf.ErrClosed`); the goroutine then closes its links and the BPF objects.

## `rawEvent` ↔ C `struct event`

```c
struct event {
    __u32 pid;
    __u32 tgid;
    char  comm[16];
    char  path[256];
};
```

```go
type rawEvent struct {
    PID  uint32
    TGID uint32
    Comm [16]byte
    Path [256]byte
}
```

Decoded via `binary.Read` with `binary.NativeEndian`. The Go side trims each `[N]byte` field at the first NUL.

## Privileges

Loading BPF programs and attaching tracepoints requires `CAP_BPF` and `CAP_PERFMON` on modern kernels (≥ 5.8), or `CAP_SYS_ADMIN` on older ones. The simplest way to run is `doas` or `sudo`. A `make setcap` target is on the roadmap so the binary can carry the caps directly.
